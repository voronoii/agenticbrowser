## 에이전틱 브라우저(크롤러) 개발

### 개발 배경

1. **복합적인 질문에 대한 최적화된 툴 매칭**: 단일 툴로 해결할 수 없는 복잡한 질문이 들어왔을 때, 여러 툴을 적절히 조합하여 답변할 수 있도록 내부 오케스트레이션이 필요. 예를 들어 "마포구 주차 좋은 아파트"는 검색 → 필터링 → 상세정보 조회의 연속적인 툴 호출이 필요함

2. **질문 커버리지 확대**: 기존에는 특정 패턴의 질문만 처리 가능했지만, 통합 에이전트를 통해 다양한 형태의 질문에 유연하게 대응 가능. 모호한 질문, 단계적 탐색이 필요한 질문, 여러 정보를 조합해야 하는 질문 등을 모두 처리

3. **툴 호출 제어권 확보**: 외부 에이전트에 의존하면 툴 호출 순서나 조합을 우리가 제어할 수 없음. 내부에 오케스트레이션 Agent를 두어 도메인 지식을 활용한 최적의 툴 실행 전략 수립 가능

### 프로젝트 목표

> 목표: 데이터 수집 전용 → 범용 브라우저 자동화 에이전트 (데이터 수집은 하위 유스케이스)

사용자가 자연어로 목표를 입력하면, AI 에이전트가 웹 브라우저를 직접 조작하여 태스크를 수행한다. 클릭, 입력, 스크롤 등 사람이 하는 브라우저 조작을 에이전트가 대신하며, 오류가 발생하면 스스로 다른 방법을 시도한다. 에이전트가 판단이 어려운 상황에서는 사람에게 질문하거나, 사람이 직접 개입할 수도 있다. (Human-in-the-Loop)

사용자 입력은 3가지로 구성된다:

| 입력 | 필수 | 설명 | 예시 |
|---|---|---|---|
| **Goal** | 필수 | 무엇을 달성할 것인가 | "네이버 뉴스에서 오늘의 헤드라인 5개 수집" |
| **Direction** | 선택 | 어떤 경로로 수행할지 힌트 | "뉴스 탭 클릭 → 헤드라인 섹션에서 수집" |
| **Context** | 선택 | 로그인 정보, 파일, 제약조건 등 | 계정 정보, 참고 CSV 파일 |

> Direction이 있으면 더 빠르고 정확하게, 없으면 에이전트가 자율적으로 탐색한다.

### 동작 방식

핵심 루프는 **Observation → Decision → Action**의 반복이다.

```
사용자가 Goal 입력
  → 브라우저가 페이지를 열고 현재 상태를 읽음 (Observation)
    → LLM이 상태를 보고 다음 행동을 결정 (Decision)
      → 브라우저에서 해당 행동을 실행 (Action)
        → 다시 상태를 읽음 → 반복
          → 태스크 완료(done) 또는 사람에게 질문(ask_human)으로 종료
```

매 스텝마다 페이지의 클릭 가능한 요소들(버튼, 링크, 입력창 등)을 자동으로 추출하고 번호를 매긴다. LLM은 이 번호 목록을 보고 "5번 버튼 클릭", "3번 입력란에 텍스트 입력" 같은 식으로 JSON 형태의 액션을 결정한다.

### 현재 구현 상태 (Phase 1-A)

**"단일 에이전트가 단일 브라우저에서 하나의 태스크를 수행하는 최소 동작 시스템"**

구현 완료된 것:
- 브라우저 자동 조작 (열기, 이동, 클릭, 입력, 스크롤 등 10가지 액션)
- DOM 기반 페이지 상태 추출 (기존 aria_snapshot 5개 → 97개 요소 추출로 개선)
- LLM 연동 (OpenAI / Anthropic 선택 가능)
- 에이전트가 같은 행동만 반복할 때 자동 감지 및 종료 (Stuck 감지)
- 팝업/쿠키 배너 등 오버레이 자동 제거 (코드 레벨 + 프롬프트 레벨 2중 방어)
- WebSocket 기반 실시간 UI (태스크 입력 → 브라우저 스크린샷 실시간 표시 → 에이전트 로그 스트리밍)
- Docker 배포 지원

### 현재 제한사항

- 단일 브라우저/단일 탭만 지원 (새 탭, 팝업 윈도우 처리 불가)
- 사람의 개입이 "정지(stop)"만 가능 (일시정지 후 지시 주입 등은 아직 미구현)
- 로그인/인증 자동화 없음 (수동으로 처리해야 함)
- 파일 업로드/다운로드 미지원
- 시각 정보(스크린샷) 활용 없음 — 텍스트 기반 상태만 사용
- 매번 처음부터 탐색 (학습/패턴 재사용 없음)

### 향후 계획

**Phase 1-B — Human-in-the-Loop + 서브에이전트 **
→ 사람이 에이전트를 일시정지하고 직접 조작하거나 지시를 줄 수 있도록
→ 역할 분리: 정찰(Scout), 실행(Executor), 인증(Auth), 검증(Validator) 등 서브에이전트 도입
→ 멀티탭 지원, 파일 처리, LLM 응답 검증 강화

**Phase 2 — 자가학습 + VLM 실험 **
→ 성공 패턴을 저장해서 반복 태스크에 재사용 (Pattern Compiler)
→ 스크린샷을 함께 활용하는 VLM(Vision Language Model) 도입 효과 검증
→ 자동화된 성능 테스트 (Evaluation Harness)

**Phase 3 — 스케일 **
→ 반복 크롤링 스케줄러, 분산 실행
→ 100만 페이지/월 처리 목표
